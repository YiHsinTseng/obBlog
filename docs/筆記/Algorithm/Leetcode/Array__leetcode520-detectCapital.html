[Array] leetcode520_detect Capital

先說結論，以下是 leetcode520 detectCapital 相對比較簡潔易懂的解法。如果想看我個人的解法思路與檢討，直接跳到後續特定段落。

解題要點
  • 先觀察輸出是否可以簡單枚舉，不要馬上迴圈用遍歷。
  • 封裝函式以增加可讀性

推薦的解法​

較簡潔的解法​

var detectCapitalUse = function(word) {
    return (word===word.toUpperCase()) || (word===word.toLowerCase()) || (word[0]===word[0].toUpperCase() &&  word.slice(1)===  word.slice(1).toLowerCase())
}

// 測試
console.log(detectCapitalUse("USA"),true);    // true
console.log(detectCapitalUse("leetcode"),true); // true
console.log(detectCapitalUse("Google"),true);   // true
console.log(detectCapitalUse("FlaG"), false);     // false

當然也可以正則表示法下手，前提必須要很熟悉正則表達式：

較炫技的解法​

var detectCapitalUse = function(word) {
    // 判斷是否符合條件
    return /^[A-Z]+$|^[a-z]+$|^[A-Z][a-z]+$/.test(word);
};

// 測試
console.log(detectCapitalUse("USA"),true);    // true
console.log(detectCapitalUse("leetcode"),true); // true
console.log(detectCapitalUse("Google"),true);   // true
console.log(detectCapitalUse("FlaG"), false);     // false
  • ^[A-Z]+$：全部大寫
  • ^[a-z]+$：全部小寫
  • ^[A-Z][a-z]+$：首字母大寫，其他小寫

解法分析​

這題思路的關鍵是分析正確輸出的可能解決方法，從而選擇較好的程式實作方式。這點是我在緊張時容易疏忽的部分。(緊張時會反射性使用遍歷迴圈的形式，而忽略是輸出其實是可以簡單枚舉)

//success (3種情況)
//大,大....大
//大,小....小
//小,小....小

//fail (其餘所有情況)

此題目來看，正確輸出成功情況可以明顯被枚舉，因此不需要遍歷所有元素。加上 Array.toUpperCase() 這個函式可以處理不只一個元素，所以這題就變成以 Array 角度來分析枚舉成功案例。

  • 全部大寫 就是用 word===word.toUpperCase()
  • 全部小寫 就是用 word===word.toLowerCase()
  • 第一個大寫 其餘小寫就是
    • 先判斷首字母 Array: word[0]===word[0].toUpperCase()
    • 再判斷後續字母構成的 Array: word.slice(1)===word.slice(1).toLowerCase()

遍歷迴圈的解法​

即便因為緊張忽略的第一個要點（枚舉少數），那在撰寫迴圈時也應該要做好分點討論。

從剛剛的分析來看，這要用迴圈寫應該會有三個主要分支 (後續大會成功、後續小會成功、都不會成功)，決定是否為成功案例。

//success (3種情況)
//大,大....大 (後續要大)
//大,小....小 (後續要小)
//小,小....小 (後續要小)

//fail (其餘所有情況)
(沒有後續了)
  • 在特定前提下，
    • 後續全都要大寫
    • 後續全都要小寫
  • 未達特定前提
    • 完全不可能成功

所以可以用 if-else 來區隔這樣的前提條件 還有一點要注意的是為了避免大小寫檢查會影響在 if-else 的可讀性，可以將大小寫檢查封裝成函式。

封裝函式增加可讀性​

function isC(char){
    if(typeof char !=="string"){return false}
    return char===char.toUpperCase()
}

if-else 形式如所示，將變成找尋失敗案例就回傳結果的形式。

按照條件建立流程分支​

var detectCapitalUse = function(word) {
    if(isC(word[0])&&isC(word[1])){ //大大
        //if 其餘(index=2)有小寫
        //  return false
    }else if((isC(word[0])&&!isC(word[1]))||(!isC(word[0])&&!isC(word[1]))){
        //大小或大大
        //if 其餘(index=2)有大寫 
        //  return false
    }else{//其餘不可能成功
        return false
    }
    return true
};

在繼續遍歷其餘元素要注意迴圈寫法最好要使用 for(let i=0;i<word.length;i++) 來指定從特定元素開始，若用 for (const w of word) 比較不好排除前面幾個元素。

if(isC(word[0])&&isC(word[1])){ //大大
    //if 其餘(index=2)有小寫
    for(let i=2;i<word.length;i++){
        if(!isC(word[i])){
            return false
        }
    }
}else if((isC(word[0])&&!isC(word[1]))||(!isC(word[0])&&!isC(word[1]))){
    //大小或大大
    //if 其餘(index=2)有大寫 
    for(let i=2;i<word.length;i++){
        if(isC(word[i])){
            return false
        }
    }

最終形式結合前兩者，如以下所示：

var detectCapitalUse = function(word) {

    function isC(char){
        if(typeof char !=="string"){return false}
        return char===char.toUpperCase()
    }
    
    if(isC(word[0])&&isC(word[1])){ //大大
        //if 其餘(index=2)有小寫
        for(let i=2;i<word.length;i++){
            if(!isC(word[i])){
                return false
            }
        }
    }else if((isC(word[0])&&!isC(word[1]))||(!isC(word[0])&&!isC(word[1]))){
        //大小或大大
        //if 其餘(index=2)有大寫 
        for(let i=2;i<word.length;i++){
            if(isC(word[i])){
                return false
            }
        }
    }else{//其餘不可能成功
        return false
    }
    return true
};

// 測試
console.log(detectCapitalUse("USA"),true);    // true
console.log(detectCapitalUse("leetcode"),true); // true
console.log(detectCapitalUse("Google"),true);   // true
console.log(detectCapitalUse("FlaG"), false);     // false

檢討要點​

  • 不要馬上就使用迴圈遍歷，先觀察正確輸出的可能解決方法是否可以簡單枚舉
  • 對於不熟悉的函式，如果容易描述過程，要獨立成函式以增加可讀性
  • 迴圈應優先選擇用 for(let i=0;i<word.length;i++) 形式，比較好控制起始點。
  • 卡住時，可以從頭開始逐一瀏覽迴圈邏輯，並根據錯誤的 test case 印出字串判斷實際運行情況與預期結果的差異。當然如果有時間能冷靜從 test case 分析開始也是不錯。
