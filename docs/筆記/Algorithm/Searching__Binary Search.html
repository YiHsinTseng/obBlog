[Searching] Binary Search

算法關鍵字

排序後，矩陣，查找。

提示

mid 中位數計算

while 迴圈條件與區間設計： left, right , while, [ l,r ] or [l,r)

binarySearch 的特點在於中位數 mid 的計算以及 while 迴圈條件設計，透過 while 迴圈不斷讓迴圈內部的數值產生變化 (mid 影響迴圈條件 left, right)，進而得出結果。

與線性全搜尋的平均 O(N) 複雜度相比，二分搜尋的平均 O(log N) 複雜度顯著提升了搜尋效率。因此只要涉及到排序後矩陣的搜索，就可以想到二分搜索。

中位數計算​

中位數的計算是二分搜索的核心，通常通過 mid = left + (right - left) / 2 計算得到。這種計算方式避免了可能的整數溢出問題（如  (left + right) / 2 可能會導致溢出）。中位數用於將當前資料集合分成兩半，從而決定下一步的查找範圍（左半部分或右半部分）。

但 JS 原則上不用考慮溢出的問題，因為 JavaScript 的 Number 類型是基於 IEEE 754 雙精度浮點數（64 位浮點數），可以表示的整數範圍非常大（從 -2^53 到 2^53 - 1）。因此，在使用 left + right 計算時，即使處理非常大的陣列索引，也不會出現溢出問題。

while 循環條件的設計​

其中 while 循環條件的設計涉及兩個面向：分別是查找條件以及查找區間。

查找條件主要體現在對 num[mid] 的判斷式中，通過該判斷式確定 left 和 right 的移動方向，二分搜索過程中，逐步縮小由 left 和 right 構成的查找區間，最終在循環結束時定位到最接近查找條件的目標位置。

而 left 與 right 兩數值建構查找區間類型，可以分為閉區間 [ left,right ] 或右半開區間 [ left,right ) ，兩種區間在循環條件差異。

我自己的習慣是以閉區間 [ left,right ]，因為比較符合循環計算中位數的想法。但如何有額外條件如找尋適當插入索引位置 (Leetcode 35)，或許右半開區間 [ left,right ) 比較直觀。

以下列出閉區間與右半開區間的寫法：

閉區間​

閉區間 [ left,right ] ，適用於目標元素明確在矩陣中，較符合二分搜索算法中計算矩陣內計算中位數的思維。此時會分別設左右指針為 left=0, right=len-1。

在二分搜索算法中，while 循環條件通常是 left <= right。循環會繼續執行，直到 left 指針大於 right 指針。這個條件確保了在資料集合的有效範圍內持續進行查找，直到找到目標元素或確定目標元素不在集合中。

let left=0
let right=nums.length-1
while(left<=right){//不在邊界內就要停止
    const mid=Math.floor((left + right) / 2);
    if(num[mid]===target){
        return mid
    }else if(num[mid]<target){//邊界比較小就要向右移
        left=mid+1
    }else{//邊界比較大就要向左移
        right=mid-1 //差異點
    }
}
備註

前提一：mid 不該被重複判斷。

前提二：mid 是否等於 target 已先被判斷，才移動 left 或 right 指針。

基於迴圈條件屬於閉區間，所以 left 指針如果要調整就要 mid+1，right 指針的位置要調整就要 mid-1，這樣才不會讓 mid 位置被重複判斷。

右半開區間​

但若是數值插入特定位置這種題目，需要考慮到 arr [len] 才行，這部分不在閉區間迴圈條件內，而設成右半開區間 [ left,right ) 代表允許的範圍不限於矩陣內部（可在矩陣右邊界），因為開區間本身 right 就是 len 而非矩陣內部的最後一個索引值。 當 left === len 時，迴圈就會停止，故可以簡化寫法將 arr[len] 表達成 arr[right]，此寫法 更適合插入節點的案例，可避免了如果 left 最終越過了 right，還需要再檢查當前 left 是否等於陣列長度，是否應該插入到末尾，這導致程式碼複雜度增加。

let left=0
let right=nums.length //差異點
while(left<right){//右半開區間
    const mid=Math.floor((left + right) / 2);
    if(num[mid]===target){
        return mid
    }else if(num[mid]<target){
        left=mid+1
    }else{
        right=mid//差異點
    }
}
return right//插入節點的案例
備註

前提一：mid 不該被重複判斷。

前提二：mid 是否等於 target 已先被判斷，才移動 left 或 right 指針。

基於迴圈條件屬於右半開區間，因此 right 指針設為 mid 也不會被判斷，故只有 left 指針如果要調整就要 mid+1。

不過其實回傳 left 或 right 都可以，因為一旦迴圈 left===right 就會停止，而 right 本身就是 len，故 left 也是 len，代表矩陣右外的索引值。

承上，其實閉區間也能處理這種插入節點的問題，但是就比較不直觀。 需要思考邊緣情況，就是 l>r 時即停止，且此時 l=r+1，也就是 l 就是 len，故回傳 l 即可。


let l=0
let r=nums.length-1//開區間
while(l<=r){
    const mid=Math.floor(l+(r-l)/2)
    if(nums[mid]===target){
        return mid
    }else if(nums[mid]<target){
        l=mid+1// 右移到最後 l>r直接停止
    }else{
        r=mid-1
    }
}
return l;
